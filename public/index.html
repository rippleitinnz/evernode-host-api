01 — Quick Start

Start querying in seconds

No authentication required. All endpoints return JSON. CORS enabled for browser requests.

# Active hosts with reputation and available slots curl "https://api.onledger.net/hosts?active=true&minRep=200&minSlots=1&limit=10" # Single host by address curl "https://api.onledger.net/hosts/rnpkkEEMDSYAg1G6eHWF66kKjrKoAqMbtV" # Search by domain, email or description curl "https://api.onledger.net/hosts/search?q=onledger" # Top hosts leaderboard curl "https://api.onledger.net/leaderboard?metric=hostReputation&limit=10" # Compare multiple hosts curl "https://api.onledger.net/hosts/compare?addresses=rXXX,rYYY,rZZZ" # Batch lookup (POST) curl -X POST "https://api.onledger.net/hosts/batch" \ -H "Content-Type: application/json" \ -d '{"addresses":["rXXX","rYYY"]}' # Heartbeat history for a host curl "https://api.onledger.net/hosts/rnpkkEEMDSYAg1G6eHWF66kKjrKoAqMbtV/history" # Network statistics curl "https://api.onledger.net/stats"
02 — Endpoints

API reference

Expand ▼
GET/hostsFiltered host list

Returns a paginated list of hosts matching the specified filters. All parameters optional. Defaults: minXah=5, minEvr=5. Responses include total, hasMore, nextOffset and prevOffset pagination fields.

ParameterTypeDefaultDescription
activeboolean—Filter by active status
minSlots / maxSlotsinteger—Available instances range
minRep / maxRepinteger—Reputation score range (0–255)
includeUnscoredbooleanfalseInclude hosts with rep=0
minXahnumber5Minimum XAH balance
minEvrnumber5Minimum EVR balance
minLease / maxLeaseinteger—Lease price range in drops
countrystring—2-letter ISO country code (DE, FR, JP...)
domainstring—Partial domain match
versionstring—Sashimono version (1.0.0, 0.12.1...)
minRam / minDiskinteger—Minimum RAM (MB) or disk (MB)
isATransfererinteger—0=exclude hosts in transfer mode, 1=only transferring hosts
reputedOnHeartbeatboolean—true=only hosts actively being reputation-tested
minAccumulatedRewardnumber—Minimum lifetime EVR rewards earned
hasDescriptionboolean—true=only hosts with a description set, false=only hosts without
hasEmailboolean—true=only hosts with an email set, false=only hosts without
hasDomainboolean—true=only hosts with a domain set, false=only hosts without
description_likestring—Partial match on description field
fieldsstring—Comma-separated list of fields to return e.g. address,domain,hostReputation
sortstring—Shorthand sort e.g. sort=hostReputation:desc (alternative to sortBy + sortDir)
sortBystringhostReputationhostReputation, availableInstances, leaseDrops, xahBalance, evrBalance, ramMb, diskMb, lastHeartbeatIndex, registrationTimestamp, accumulatedReward, lastUpdated
sortDirstringdescasc or desc
limit / offsetinteger100 / 0Pagination
GET/hosts/searchFull text search

Search across domain, email and description fields. Returns hosts sorted by reputation. At least one of q, hasDescription or description_like is required.

ParameterTypeDefaultDescription
qstring—Search term (min 2 chars) — matches domain, email and description. Optional if hasDescription or description_like provided.
hasDescriptionboolean—true=only hosts with a description set
description_likestring—Partial match on description only
activeboolean—Filter by active status
fieldsstring—Comma-separated fields to return
limitinteger50Max results (up to 200)
GET/hosts/expiringSilent / stale hosts

Returns active hosts that have not sent a heartbeat within the specified number of hours.

ParameterTypeDefaultDescription
hoursinteger24Silence threshold in hours
fieldsstring—Comma-separated fields to return
limitinteger50Max results (up to 200)
GET/hosts/compareCompare multiple hosts

Fetch full data for multiple hosts in a single request, returned in the same order as requested. Max 20 addresses.

ParameterTypeDefaultDescription
addressesstring—Comma-separated XRPL addresses (max 20)
fieldsstring—Comma-separated fields to return
GET/leaderboardTop hosts by metric

Returns the top N active hosts ranked by the specified metric.

ParameterTypeDefaultDescription
metricstringhostReputationhostReputation, accumulatedReward, xahBalance, evrBalance, ramMb, diskMb, availableInstances
fieldsstring—Comma-separated fields to return
limitinteger20Number of results (max 100)
POST/hosts/batchBulk address lookup

Fetch data for up to 100 hosts in a single POST. Body must be JSON with an addresses array. Optionally include a fields array to limit returned fields.

{ "addresses": ["rXXX...", "rYYY..."], "fields": ["address", "domain", "hostReputation"] }
GET/hosts/:addressSingle host full data

Returns complete data for a single host by XRPL address.

ParameterTypeDefaultDescription
fieldsstring—Comma-separated fields to return
GET/hosts/:address/historyHeartbeat history

Returns a time-series of heartbeat snapshots capturing reputation, slots, balances and lease price over time. History accumulates from API v1.2.0 onwards. Supports pagination.

ParameterTypeDefaultDescription
limitinteger100Number of snapshots to return (max 500)
offsetinteger0Pagination offset
GET/hosts/randomRandom host sample

Returns a randomly selected set of hosts matching the given filters. Useful for load distribution without client-side shuffling.

ParameterTypeDefaultDescription
countinteger10Number of random hosts to return (max 200)
minRepinteger—Minimum reputation score
minSlotsinteger—Minimum available slots
countrystring—Filter by country code
activebooleantrueFilter by active status
fieldsstring—Comma-separated fields to return
GET/countriesHost counts by country

Returns all countries with registered hosts, including total host count, active host count and available slots per country.

GET/versionsSashimono version distribution

Returns all active Sashimono versions with host counts.

GET/statsNetwork summary statistics

Network-wide statistics: total and active host counts, available slots, average reputation and balances, top countries, version distribution and count of hosts with descriptions.

GET/healthAPI health check

Returns API status, host count, history snapshot count, last full scan timestamp and whether a scan is currently in progress.

Common queries shown in curl, browser fetch, and Node.js. All examples target https://api.onledger.net

Find active hosts with available slots
The most common query — active hosts with at least one free slot, sorted by reputation.
curl "https://api.onledger.net/hosts?active=true&minSlots=1&limit=20"
const res = await fetch('https://api.onledger.net/hosts?active=true&minSlots=1&limit=20'); const { hosts, total } = await res.json(); console.log(`${total} matching hosts`); hosts.forEach(h => console.log(h.address, h.availableInstances));
const { hosts } = await fetch('https://api.onledger.net/hosts?active=true&minSlots=1&limit=20').then(r => r.json()); console.log(hosts.map(h => h.address));

Batch lookup by address
POST an array of addresses to fetch up to 100 hosts in a single request — efficient for cluster managers.
curl -X POST "https://api.onledger.net/hosts/batch" \ -H "Content-Type: application/json" \ -d '{"addresses":["rXXX","rYYY","rZZZ"]}'
const { hosts } = await fetch('https://api.onledger.net/hosts/batch', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ addresses: ['rXXX', 'rYYY', 'rZZZ'] }) }).then(r => r.json()); console.log(hosts);
const myNodes = ['rXXX', 'rYYY', 'rZZZ']; const { hosts, found } = await fetch('https://api.onledger.net/hosts/batch', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ addresses: myNodes }) }).then(r => r.json()); console.log(`Found ${found} of ${myNodes.length} hosts`);

const addresses = ['rXXX', 'rYYY', 'rZZZ']; const { hosts } = await fetch(`https://api.onledger.net/hosts/compare?addresses=${addresses.join(',')}`).then(r => r.json()); console.log(hosts);
Search by domain, email or description
Full text search across operator-provided fields. Use description_like for description-only matching, or hasDescription to find all hosts that have filled one in.
# Broad search across all text fields curl "https://api.onledger.net/hosts/search?q=onledger&active=true" # Only hosts with descriptions containing "Frankfurt" curl "https://api.onledger.net/hosts/search?description_like=Frankfurt&active=true" # All active hosts that have set a description curl "https://api.onledger.net/hosts/search?hasDescription=true&active=true"
const { hosts, total } = await fetch('https://api.onledger.net/hosts/search?q=onledger&active=true').then(r => r.json()); console.log(`${total} results`, hosts);
const { hosts, total } = await fetch('https://api.onledger.net/hosts/search?hasDescription=true&active=true&limit=200').then(r => r.json()); console.log(`${total} hosts have descriptions`); console.log(hosts.map(h => ({ address: h.address, description: h.description })));
Paginate through all hosts
Use the hasMore and nextOffset fields in the response to page through results efficiently.
# Page 1 curl "https://api.onledger.net/hosts?active=true&limit=100&offset=0" # Use nextOffset from response for page 2 curl "https://api.onledger.net/hosts?active=true&limit=100&offset=100"
async function getAllHosts() { let offset = 0, all = [], hasMore = true; while (hasMore) { const data = await fetch( `https://api.onledger.net/hosts?active=true&limit=100&offset=${offset}` ).then(r => r.json()); all = all.concat(data.hosts); hasMore = data.hasMore; offset = data.nextOffset; } return all; }
async function getAllHosts() { let offset = 0, all = [], hasMore = true; while (hasMore) { const { hosts, hasMore: more, nextOffset } = await fetch( `https://api.onledger.net/hosts?active=true&limit=100&offset=${offset}` ).then(r => r.json()); all = all.concat(hosts); hasMore = more; offset = nextOffset; } console.log(`Fetched ${all.length} hosts`); return all; } getAllHosts();
Top hosts leaderboard
Rank active hosts by reputation, earnings, resources or available slots.

const { hosts, label } = await fetch('https://api.onledger.net/leaderboard?metric=accumulatedReward&limit=10').then(r => r.json()); console.log(label, hosts.map(h => ({ address: h.address, reward: h.accumulatedReward })));
const metrics = ['hostReputation', 'accumulatedReward', 'ramMb']; const results = await Promise.all( metrics.map(m => fetch(`https://api.onledger.net/leaderboard?metric=${m}&limit=5`).then(r => r.json())) ); results.forEach(r => console.log(r.label, r.hosts.map(h => h.address)));
Host heartbeat history
Track reputation, slots and balance changes over time for a specific host.
curl "https://api.onledger.net/hosts/rnpkkEEMDSYAg1G6eHWF66kKjrKoAqMbtV/history?limit=50"
const address = 'rnpkkEEMDSYAg1G6eHWF66kKjrKoAqMbtV'; const { history } = await fetch(`https://api.onledger.net/hosts/${address}/history?limit=50`).then(r => r.json()); history.forEach(h => console.log(new Date(h.timestamp).toISOString(), h.hostReputation));
const address = 'rnpkkEEMDSYAg1G6eHWF66kKjrKoAqMbtV'; const { history, domain } = await fetch(`https://api.onledger.net/hosts/${address}/history`).then(r => r.json()); console.log(`${domain}: ${history.length} snapshots`); console.log(history);
